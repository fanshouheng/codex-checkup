# 报告解释指南

## 优先级

- `P0`：敏感信息暴露、明显破坏性行为等应立即处理的风险
- `P1`：高概率影响安全、返工或交付的风险
- `P2`：应安排处理的流程、维护或效率问题
- `P3`：低风险观察项，先验证再行动

## 置信度

- `高`：来自结构化配置、文件存在性或 Git 等可直接验证事实
- `中`：来自稳定但不完备的模式和足够样本
- `低`：本地格式不稳定、样本少或只能进行间接推断

## 怎样给建议

先处理少量高影响问题。每项建议应说明：先做什么、保留什么能力、如何验证、是否需要批准。没有项目级证据时，不要说项目“路子错了”；应说当前证据只能支持哪一种流程风险。

## 覆盖状态

- `complete`：该模块按预期读取并完成检查
- `partial`：只读取到部分来源或遇到未知格式
- `unavailable`：来源不存在、权限不足或运行环境不支持
- `failed`：模块执行错误，报告应保留错误类别但不泄露敏感数据
