# 隐私与数据处理

## 数据最小化

基础扫描器只读取完成检查所需的最少字段：配置键、Skill 元数据、会话角色与统计信号、项目路径映射和 Git 摘要。

报告不得包含：

- 用户或助手聊天原文
- 配置值中的密钥、令牌、密码或完整环境变量
- 完整用户主目录和完整项目绝对路径
- `.env`、SSH 私钥、系统凭据文件内容
- 完整 MCP schema、工具参数或插件缓存内容

## 深度协作诊断

深度诊断会另外生成 `.codex-health-private/collaboration-evidence.json`。它保存少量顺利完成样本和摩擦事件前后的短消息，并对常见密钥、用户目录、Codex 目录和项目路径做尽力脱敏。

该文件不是可分享报告，仍可能包含业务内容或脱敏器未识别的信息。读取它会让片段进入当前 Codex 上下文：

- 用户明确要求全面体检、全景体检，或审核聊天、协作、返工时，可视为对本次最小化读取的授权。
- 用户只要求普通设置检查时，读取前先说明范围并征得同意。
- 最终回复应概括模式，不复制大段原文。
- 不上传、提交或自动分享私有证据包。

## 路径脱敏

Codex 主目录显示为 `$CODEX_HOME`，用户主目录显示为 `$HOME`，当前项目显示为 `$PROJECT`。其他项目只显示目录名和短哈希，用于区分同名目录。

## 本地处理

脚本不发起网络请求，不调用外部模型，不安装依赖。基础报告写入用户指定的输出目录，私有证据包写入独立的已忽略目录。任何上传、分享或跨机器汇总都需要用户另行明确授权。

## 修复边界

体检和修复是两个阶段。体检阶段不写 Codex 主目录。修复阶段需要：

1. 明确要改的文件和原因
2. 生成备份
3. 展示最小差异
4. 获得用户确认
5. 修改后按同口径复测
